⚡ Workflow Sequence

1
Planning Sequential
🏛️ Architect → Orchestrator Produces system architecture, component breakdown, data flow, IoC DAL design doc/HLD.md
🔧 Tech Lead → Orchestrator Full package layout, entity specs, @Query methods, enrichment formula, test matrix, Docker doc/TECH_DESIGN.md
2
Design Review Parallel fan-out
🧪 QA / Chaos Engineer
  • TOCTOU race on RepeatOffenderCache
  • Batch 501 → no upper bound check
  • Future timestamp injection
  • Stats range unbounded
  • Log injection via ruleMessage
🔐 Security Specialist
  • JPQL injection via string concat
  • X-Forwarded-For spoofing
  • H2 console exposed in prod
  • No rate limiting on ingest
  • Root user in Docker
QA + Security → Tech Lead 12 findings consolidated → Tech Design updated with mitigations mapped to stories
3
Sprint Planning Sequential
📋 Team Lead → Backlog 17 user stories across 7 sprints. QA/security findings mapped to acceptance criteria. doc/SPRINT_PLAN.md
S1: Domain & DAL S2: Ingestion S3: Enrichment S4: Stats+Samples S5: Alerting S6: Web+Docker S7: Tests
4
Development Loop Parallel when possible
📋 Team Lead → 👨‍💻 Senior Dev Complex stories: EnrichedEvent entity, EventRepository @Query, EventIngestionService, StatsService, AlertService, Controllers, RateLimiter
📋 Team Lead → 👩‍💻 Junior Dev Simpler stories: enums, DTOs, projections, CacheConfig, DataGenerator, test stubs, Dockerfile
Dev → 🧪 QA Story delivered → QA reviews → pass/fail → dev picks next story from backlog
Per-story git commit → milestone tag at sprint boundary
5
Code Review Loop Loop until clean · cap $300
🔐 Security + 🧪 QA → Codebase System-wide review: imports, null-safety, validation, JPQL safety, log injection, Docker user
Findings → 👨‍💻 Senior Dev Fix all findings → re-run review → repeat until 0 findings
